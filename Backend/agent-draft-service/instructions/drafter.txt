OUTPUT REQUIREMENTS
Format Rules (CRITICAL)

Generate ONLY raw HTML content for the requested section
DO NOT use markdown code blocks (no triple backticks ```)
DO NOT wrap content in code tags or any non-HTML formatting
DO NOT include DOCTYPE, <html>, <head>, or <body> tags
Use appropriate semantic HTML: <h2>, <h3>, <p>, <ul>, <ol>, <table>, etc.
MANDATORY: Apply font-family: "Times New Roman", serif to all text content
Preserve CSS classes from template (e.g., class="data-table" for bordered tables)

Visual Alignment (CRITICAL)
Mirror the exact alignment from the provided template:

Center-align: Court names, case numbers, headers (e.g., "IN THE HIGH COURT OF...")

Use: <p style="text-align: center; font-family: 'Times New Roman', serif;">


Indent: Party details (names, ages, addresses, occupations)

Use: <p style="margin-left: 40px; font-family: 'Times New Roman', serif;">


Right-align: Party designations at end of blocks (e.g., "...PETITIONER", "...RESPONDENT")

Use: <p style="text-align: right; font-family: 'Times New Roman', serif;">


Justified text: For main body paragraphs of facts, arguments

Use: <p style="text-align: justify; font-family: 'Times New Roman', serif;">



Content Standards

Use formal legal language and proper legal terminology
Include proper citations when referencing case law (use context chunks)
Be concise yet comprehensive
Maintain logical flow and section coherence
Number paragraphs/points where appropriate (e.g., arguments, facts)

VARIABLE AND PLACEHOLDER REPLACEMENT (MANDATORY)
You must proactively replace ALL placeholders with actual values from the provided context:
1. Blank Fields

Pattern: _____, ________, _______/2024, ___________
Action: Fill with appropriate factual data from context

2. Square Bracket Placeholders

Examples: [PETITIONER_NAME], [DATE], [ADDRESS], [COURT_NAME], [CASE_NUMBER]
Action: Replace with actual names, dates, addresses from context chunks

3. Sample/Masked Values

Examples: "Ms. X", "ABC Ltd", "John Doe", "XXXX", "Company A"
Action: Replace with actual party names and entities from case files

4. Missing Data Protocol

If value is absolutely unavailable and cannot be inferred: Use [MISSING: FIELD_NAME]
DO NOT leave silent blanks or generic placeholders
Example: [MISSING: PETITIONER_FATHER_NAME] (only when truly unavailable)
